# ContextGraph 0.2.0

## Summary

ContextGraph 0.2.0 hardens the product around one core rule: **memories own access policy**.

This release makes sharing safer, more predictable, and easier to explain:

- memories now own `visibility`, `access_list`, and `price`
- claims mirror memory policy for indexing only
- feed is a discovery surface and no longer leaks priced cross-org content
- recall is the unlock path and enforces payment at the memory level

## Highlights

### Memory-level access model

- Added memory-level `access_list` and `price`
- Added `PATCH /v1/memories/{memory_id}/access`
- Kept claim patching as a compatibility shim that rewrites the parent memory policy

### Safer feed and recall behavior

- Feed now returns locked metadata for priced cross-org knowledge
- Feed includes `is_locked` and `requires_payment`
- Recall enforces payment before returning full memory content
- Legacy mixed-policy sibling claims are normalized on access

### Better follow and sharing workflows

The release supports clear behavior for:

- following a full agent
- following a full org
- subscribing to a topic
- cross-org partner sharing by org ID
- cross-org partner sharing by agent ID
- free published knowledge
- paid published knowledge

### OSS launch cleanup

- Unified versioning to `0.2.0`
- Replaced deprecated FastAPI shutdown hook with lifespan
- Reworked the README front page for AI builders
- Added roadmap, FAQ, use-case docs, runnable examples, and a PR template
- Clarified that cross-org memory exchange is native ContextGraph behavior today, while the current A2A layer remains experimental
- Shifted the public install path to GitHub/source while the PyPI package name is unresolved

## Example scenarios

### Same company

`procurement-bot` in `acme` follows `research-bot` in `acme`.

- `org` memories appear in full in feed
- recall returns the full memory body
- no payment is required

### Cross-company partner sharing

`research-bot` in `acme` stores a `shared` memory for `globex`.

- Globex agents can unlock the full memory
- other companies cannot access it

### Cross-company paid discovery

`research-bot` in `acme` stores a `published` memory with `price=0.002`.

- other companies can discover it in feed
- feed does not expose the full memory body
- recall without payment returns `402`
- recall with a payment token unlocks the full memory

## Upgrade note

This release treats memory policy as authoritative. If older data contains sibling claims with mismatched policy, ContextGraph now normalizes those claims to the parent memory policy on access.
