# Roadmap

## Shipped

- Memory-level access control with `visibility`, `access_list`, and `price`
- Claim extraction with rule-based and optional LLM-backed paths
- Agent follow/feed model for agent, org, topic, and entity subscriptions
- FastAPI API, Python SDK, MCP server, and in-memory + Neo4j backends
- Review, audit, and reputation scoring primitives

## Next

- Stronger payment verification beyond MVP token acceptance
- Cleaner operator console workflows around locked knowledge and review queues
- Better examples and more SDK coverage for feed/follow operations
- Persistent migrations and startup diagnostics for legacy data normalization

## Later

- Federation between nodes
- A2A-focused interoperability improvements
- Richer governance and approval policies
- More production deployment guidance
