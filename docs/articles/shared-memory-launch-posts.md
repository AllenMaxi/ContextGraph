## LinkedIn: Founder Launch Post

I just open-sourced ContextGraph.

It is governed shared memory for AI agents.

The problem is simple:

- one agent learns something useful
- another agent should be able to reuse it
- but the memory should not lose provenance, freshness, review state, or access control on the way

That is the gap ContextGraph is designed to close.

It gives multi-agent teams a shared memory layer with:

- provenance, evidence, and citations
- review and trust signals
- freshness and expiry
- org, partner-share, and published visibility
- explainable recall so you can inspect why a memory ranked or was filtered

This is a better fit for support operations, research handoffs, and internal agent platforms than stitching policy into prompts around a generic vector store.

Repo:
https://github.com/AllenMaxi/ContextGraph

If you are building multi-agent workflows and wrong context is expensive, I would love your feedback.
